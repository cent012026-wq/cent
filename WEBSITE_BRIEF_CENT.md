# cent: Brief Completo Para Crear La Web Comercial

## 1. Objetivo de este documento

Este documento es la fuente de verdad para que un agente como Claude diseñe y redacte una página web comercial para `cent`.

La web debe:

1. Explicar con claridad qué es `cent`.
2. Mostrar el problema que resuelve para comercios pequeños y medianos.
3. Convertir visitantes en leads, demos o registros.
4. Comunicar una imagen de SaaS moderno, serio y confiable.
5. Mantener claims realistas, alineados con el producto actual.

Este documento no es un README técnico. Es un brief de negocio, marketing, UX y contenido para construir una web pública.

---

## 2. Qué es cent

`cent` es un SaaS para comercios que permite registrar ventas y gastos por WhatsApp y ver el negocio desde un panel web administrativo.

La idea central del producto es simple:

> El comercio no debería depender de libretas, notas sueltas o Excel desordenado para entender cuánto vendió, cuánto gastó y cómo va su caja.

Con `cent`, el negocio puede escribir por WhatsApp cosas como:

- "Vendí 2 camisas a 80 mil"
- "Gasté 25 mil en transporte"
- "Cómo vamos hoy"

Y el sistema:

1. interpreta el mensaje,
2. registra la transacción,
3. organiza la información del negocio,
4. responde con métricas o confirmaciones,
5. permite ver todo desde la web.

## 3. Resumen ejecutivo del negocio

### Propuesta de valor en una frase

`cent` convierte WhatsApp en la forma más simple de registrar y entender las ventas y gastos de un comercio.

### Propuesta de valor extendida

`cent` ayuda a comercios que ya trabajan en WhatsApp a operar mejor sin obligarlos a aprender software complejo. En lugar de cambiar el hábito del negocio, se apoya en el canal que ya usan todos los días y lo conecta con un panel administrativo claro, móvil y enfocado en operación real.

### Categoría del producto

- SaaS B2B
- Software operativo para pequeños comercios
- Finanzas operativas ligeras
- WhatsApp-first business software

### Etapa del producto

Producto temprano / V1 funcional.

Eso significa que la web debe verse profesional, pero no debe inventar funcionalidades de ERP, contabilidad avanzada, facturación electrónica, pagos, POS, inventario completo o forecasting si no están realmente en el alcance.

## 4. Problema que resuelve

### Problema principal

Muchos comercios venden todos los días, pero no saben con precisión:

- cuánto vendieron hoy,
- cuánto gastaron,
- cuál es su margen,
- qué vendedor está moviendo más ventas,
- si van bien o mal frente a una meta,
- ni en qué se les está yendo el dinero.

### Problemas específicos del usuario

1. El negocio opera por WhatsApp, pero el control administrativo sigue siendo manual.
2. Registrar cada venta o gasto en software tradicional se siente lento o complicado.
3. Los dueños terminan revisando múltiples chats, notas y papeles.
4. Los vendedores reportan por mensajes desordenados.
5. Al final del día, nadie tiene una foto clara del negocio.
6. Las herramientas empresariales existentes suelen ser demasiado pesadas para el nivel de madurez del comercio.

### Coste del problema

- Pérdida de visibilidad diaria.
- Decisiones tardías.
- Fugas de dinero no detectadas.
- Caos operativo.
- Dependencia de una sola persona para entender el negocio.
- Imposibilidad de crecer con orden.

## 8. Mensaje central y narrativa

## Idea narrativa principal

Los comercios ya operan por WhatsApp. El problema no es el canal; el problema es que la información queda desordenada. `cent` convierte ese mismo flujo diario en datos útiles para administrar mejor el negocio.

## Estructura narrativa ideal

1. El negocio ya usa WhatsApp.
2. Pero registrar y entender la operación sigue siendo caótico.
3. `cent` convierte mensajes cotidianos en control operativo.
4. El dueño obtiene una vista clara del negocio sin agregar fricción al equipo.
5. Resultado: menos caos, más visibilidad, mejores decisiones.

## Frases de posicionamiento posibles

- "Tu negocio ya vive en WhatsApp. Ahora también puede controlarse desde ahí."
- "Registra ventas y gastos por WhatsApp. Entiende tu negocio desde una sola vista."
- "Menos libreta. Menos caos. Más control."
- "La forma simple de darle orden a la operación diaria de tu comercio."

- "Tiempo real": usar solo si el contexto es razonable.
- "Seguridad": hablar de buenas prácticas y control, no de cumplimiento no demostrado.

## 12. Diferenciadores competitivos

## Diferenciador 1: WhatsApp-first

La mayoría de herramientas obligan al usuario a entrar a un panel para operar. `cent` parte del canal que el comercio ya usa.

## Diferenciador 2: Bajo esfuerzo de adopción

No intenta cambiar hábitos primero. Se monta sobre el comportamiento real del negocio.

## Diferenciador 3: Enfoque operativo

No vende una promesa abstracta de transformación digital; resuelve un problema diario y concreto.

## Diferenciador 4: Web administrativa separada

La operación ocurre en WhatsApp y la configuración/administración en web. Esa separación es clara y fácil de comunicar.

Funciona mejor para comercios pequeños y medianos que ya operan por WhatsApp.

## Requisitos obligatorios

1. Hero con promesa clara arriba del fold.

5. Prueba visual del producto.

9. Accesibilidad razonable.
10. Carga rápida y layout limpio.
11. FAQ para objeciones.

## Buenas prácticas de contenido

1. Headline orientado a resultado.
2. Subheadline específica.
3. Beneficios antes que detalles técnicos.
4. Usar ejemplos reales de uso.
5. Incluir una narrativa clara de "problema -> solución -> resultado".

1. CTA principal único.
2. Fricción mínima en formularios.

4. FAQ cerca del final.
5. Reforzar para quién es y para quién no es.

2. Hacer evidente la relación entre WhatsApp y panel web.
